## 1. Target

* Closes #80

Two navigation defects for the Style Guide in the VitePress handbook
(config/docs only — no Style Guide rule content changed).

## 2. Specification / Test Plan

**Problem.**
1. The Style Guide showed **expanded by default**: in `doc/.vitepress/config.mts`
   the parent *Development Guide* sidebar group was `collapsed: false`, so its
   nested Style Guide sub-tree opened on load.
2. **No in-content link** to the Style Guide from its parent page
   (`doc/development-guide.md`) — reachable only via the sidebar.

**Change.**
- `config.mts`: *Development Guide* group → `collapsed: true` (the nested Style
  Guide group was already `collapsed: true`).
- `development-guide.md` §1-4 Editor Configuration: add an in-content link to the
  [Style Guide](/style-guide).

**Verification (quality gate).**
- `pnpm docs:build` passes with no dead links. ✅
- `pnpm docs:dev`: the Style Guide sub-tree is collapsed by default, and the
  Development Guide page shows a working link to the Style Guide. *(local visual
  check)*
